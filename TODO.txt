## BUG: "Sign in with Google" button overlaps Android status bar icons

### Diagnostic

Le problème vient de la combinaison de deux choses :

1. `index.html` utilise `viewport-fit=cover` dans la balise meta viewport :
   ```
   content="viewport-fit=cover, width=device-width, ..."
   ```
   → Cela permet au contenu web de s'étendre jusqu'aux bords physiques de l'écran,
     y compris DERRIÈRE la status bar Android (wifi, batterie, heure…).

2. `.home-nav` dans `home.page.scss` (ligne 88) n'utilise qu'un padding fixe :
   ```scss
   padding: 16px 20px;
   ```
   → Aucun env(safe-area-inset-top) n'est appliqué, donc le bouton "Sign in with
     Google" se retrouve visuellement dans la zone réservée à la status bar Android.

Note : le game screen utilise déjà correctement env(safe-area-inset-bottom) pour
le dashboard (table.component.scss), mais le home screen oublie le haut.

### Deux options possibles

─── Option A : CSS safe-area-inset-top (RECOMMANDÉE - 1 ligne de code) ────────

Modifier `.home-nav` dans `frontend/src/app/home/home.page.scss` :

AVANT (ligne 88) :
  padding: 16px 20px;

APRÈS :
  padding: calc(16px + env(safe-area-inset-top)) 20px 16px 20px;

→ Sur Android, env(safe-area-inset-top) retourne la hauteur de la status bar
  (généralement ~24-32dp). Sur les appareils sans encoche/status bar custom, la
  valeur est 0 donc aucun effet indésirable.
→ C'est la même approche que celle déjà utilisée pour le bas de l'écran dans
  table.component.scss.

─── Option B : Capacitor StatusBar plugin (plus de contrôle, plus de code) ────

Le plugin `@capacitor/status-bar` est déjà installé (package.json).
On pourrait appeler StatusBar.setOverlaysWebView({ overlay: false }) au démarrage
de l'app, ce qui ferait descendre automatiquement tout le contenu sous la status bar.
Inconvénient : ça change le comportement global de l'app et peut casser d'autres
écrans (ex : le game screen qui a son propre fond sombre full-screen).

### Plan d'action (Option A)

1. Ouvrir : frontend/src/app/home/home.page.scss
2. Ligne 88, remplacer :
     padding: 16px 20px;
   par :
     padding: calc(16px + env(safe-area-inset-top)) 20px 16px 20px;
3. Tester sur Android physique et vérifier que le bouton est bien sous les icônes
   de la status bar.
4. Vérifier aussi sur iOS que l'encoche ne pose pas de problème (même fix, même var).

### Fichiers concernés

- frontend/src/app/home/home.page.scss  → ligne 88  (LE FIX)
- frontend/src/index.html               → ligne 11   (cause : viewport-fit=cover)
- frontend/src/app/home/home.page.html  → lignes 5-31 (structure du nav)
