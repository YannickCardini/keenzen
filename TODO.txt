[3] Freeze the timer after the player confirms or discards their action
------------------------------------------------------------------------------
Context:
  - In table.component.ts, `startTimer()` runs a `setInterval` that ticks every
    second. It is only cleared on `onTimeUp()` or when a new turn starts via
    `gameStateService.newTurn.subscribe(() => startTimer())`.
  - `onConfirmOrDiscard()` (table.component.ts) calls `gameStateService.playAction()`
    which sends the move to the server, but does NOT stop or freeze the timer.
  - The countdown sound (`playCountdownTick`) keeps playing even after the player
    has already sent their action (lines 253–255).
  - The server controls the actual turn timeout independently; the client timer
    is only a local visual aid.

Tasks:
  a. In `onConfirmOrDiscard()`, after calling `gameStateService.playAction()`:
       - Call `clearTimer()` to stop the setInterval.
       - Leave `timeLeft` signal at its current frozen value (do NOT reset to 0).
       - This visually freezes the SVG arc at the current position.
  b. Ensure `playCountdownTick()` is no longer called after the action is sent
     (clearing the interval is sufficient since the sound is triggered inside
     the interval callback).
  c. The timer will restart naturally when the backend sends the next turn and
     `gameStateService.newTurn` emits, which already calls `startTimer()`.
  d. No server-side changes are needed.

[6] Tie the timer animation to the current-turn player
------------------------------------------------------------------------------
Context:
  - The SVG ring timer in table.component.ts (`timerCircumference`, `timerDashOffset`,
    `timerColor`) currently always displays when a turn is active, regardless of
    whose turn it is. The player must infer from context which player is active.
  - `gameStateService.isMyTurn()` returns true only when it is the local user's
    turn. `displayedGameData()?.gameState.currentTurn` holds the active player's
    color.
  - board.component.html has player-info panels placed at each side of the board
    via `PLAYER_INFO_STARTS` positions.
  - The proposed design (from the original note):
      • On the board: show an animated circular arc/ring rotating around the
        player-info panel of the CURRENT TURN player (all 4 players' turns).
      • In the table component: show/animate the SVG ring timer ONLY when it is
        the LOCAL user's turn. Grey it out or hide it during other players' turns.

Tasks:
  a. In board.component.html/scss:
       - For each player-info cell, conditionally add a CSS animation class
         (e.g., `[class.active-turn-ring]="isCurrentTurn(player.color)"`) that
         renders a rotating arc or pulsing ring around the panel.
       - The animation should use `gameStateService.data()?.gameState.currentTurn`
         to determine which panel gets the ring.
       - The ring should NOT show any countdown number; it is purely decorative/
         indicative. Use a CSS animation (e.g., rotating border or conic-gradient)
         rather than the SVG timer logic.
  b. In table.component.html/scss:
       - Wrap the SVG ring timer with `*ngIf="gameStateService.isMyTurn()"` OR
         apply a `[class.timer-inactive]` class when it is NOT the local player's
         turn, visually dimming/greying the timer.
       - The countdown sound (`playCountdownTick`) already plays only on the
         local player's turn (line 253 — `isMyTurn()` check is already in place).
       - The SVG arc animation (`timerDashOffset`, `timerColor`) can remain as-is
         but should only be prominent/colored during the local player's turn.
  c. Coordinate the two animations: the board ring indicates whose turn it is
     globally; the table timer tells the local player how much time they have
     left. They serve different purposes and can coexist.
