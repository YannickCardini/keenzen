## TODO — Feature: "Play Now" Matchmaking

Quand un joueur clique "Play Now", il rejoint la première session en attente.
Si aucune session n'existe, il en crée une. Les joueurs suivants rejoignent
la même session jusqu'à 4. Après 30 s d'attente, les slots vides sont remplis
par des bots et la partie démarre.

---

### 1. Shared types — `packages/shared/src/types.ts`

Ajouter deux nouveaux types de messages :

**Client → Serveur :**
```ts
interface JoinMatchmakingMessage {
  type: 'joinMatchmaking';
  playerName?: string; // optionnel, fallback "Player"
}
```
L'ajouter à l'union `ClientMessage`.

**Serveur → Client :**
```ts
interface MatchmakingStatusMessage {
  type: 'matchmakingStatus';
  connectedCount: number;   // combien de joueurs humains dans la session
  totalNeeded: number;      // toujours 4
  myColor: MarbleColor;     // couleur assignée à CE joueur
}
```
L'ajouter à l'union `ServerMessage`.


---

### 2. Backend — NOUVEAU fichier `backend/src/session/matchmaking-manager.ts`

Créer la classe `MatchmakingManager`. Elle gère UNE session en attente à la fois
(première arrivée, première servie). Quand la partie démarre, `session = null`
et le prochain `joinQueue()` crée une nouvelle session.

**Structure interne :**
```ts
interface PendingMatchmaking {
  messenger: MultiWsMessenger;
  players: Array<{ ws: WebSocket; color: MarbleColor; name: string }>;
  timer: NodeJS.Timeout | null;
}
```

**Couleurs assignées dans l'ordre :** red → green → blue → orange
(1er joueur = red, 2e = green, etc.)

**Méthode `joinQueue(ws, playerName = 'Player')`:**
1. Si `session === null` → créer une nouvelle session (nouveau `MultiWsMessenger`, tableau vide, timer null)
2. Assigner la couleur = `COLORS[session.players.length]`
3. `session.messenger.addConnection(color, ws)`
4. Pusher dans `session.players`
5. `broadcastStatus()` → envoyer `matchmakingStatus` à chaque joueur connecté avec son `myColor`
6. Si c'est le 1er joueur → démarrer timer de 30 s → `setTimeout(() => fillWithBots(), 30_000)`
7. Si `session.players.length === 4` → appeler `launch()` immédiatement

**Méthode `fillWithBots()` (appelée par le timer) :**
- Si `session === null` → rien à faire (déjà lancée)
- Appeler `launch()`

**Méthode `launch()` (privée) :**
1. `clearTimeout(session.timer)`
2. Construire le `GameConfig` :
   - Pour chaque couleur dans `['red','green','blue','orange']` :
     - Si cette couleur est dans `session.players` → `isHuman: true`, `name: player.name`
     - Sinon → `isHuman: false`, `name: 'Bot'`
3. Sauvegarder `messenger` dans une variable locale
4. `session = null` → prêt pour la prochaine session
5. `new Game(config, messenger)`

**Méthode `broadcastStatus()` (privée) :**
- Pour chaque joueur dans `session.players` :
  - Envoyer via `player.ws.send()` un `MatchmakingStatusMessage` avec :
    - `connectedCount: session.players.length`
    - `totalNeeded: 4`
    - `myColor: player.color`

**Gestion de la déconnexion :**
- Lors du `joinQueue()`, écouter l'événement `close` du WebSocket.
- À la fermeture : retirer le joueur de `session.players`, `broadcastStatus()`.
- Si `session.players.length === 0` → `clearTimeout(timer)`, `session = null`.
- Si au moins un joueur reste → continuer à attendre (le timer continue).

### 6. Frontend — MODIFIER `frontend/src/app/home/home.page.ts`

**Injecter `GameStateService` et `Router` dans le constructeur.**

**Ajouter les états :**
```ts
showMatchmaking = false;
matchmakingConnected = 0;   // nombre de joueurs actuellement dans la session
myMatchmakingColor: MarbleColor | null = null;
private matchmakingSub: Subscription | null = null;
```

**Méthode `openMatchmaking()` :**
1. `showMatchmaking = true`
2. `gameStateService.connect(environment.wsUrl, () => { gameStateService.sendJoinMatchmaking('Player') })`
3. Souscrire à `gameStateService.matchmakingStatus$` :
   - Mettre à jour `matchmakingConnected` et `myMatchmakingColor`
4. Souscrire à `gameStateService.data` (signal ou Observable) :
   - Dès qu'un `gameState` arrive (partie démarrée) → appeler `gameStateService.setConfig(...)` avec la config reçue et `myMatchmakingColor`, puis `router.navigate(['/game'])`

**Méthode `cancelMatchmaking()` :**
1. `matchmakingSub?.unsubscribe()`
2. `gameStateService.disconnect()`
3. `showMatchmaking = false`
4. `matchmakingConnected = 0`

**Note sur `setConfig()` :**
`GameStateService.setConfig()` sert à enregistrer la couleur du joueur local.
En mode matchmaking, appeler :
```ts
gameStateService.myPlayerColor.set(this.myMatchmakingColor);
```
(ou adapter selon comment `setConfig` fonctionne actuellement)

**Connecter le bouton "Play Now" dans le HTML :**
```html

```
