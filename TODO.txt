### 4. Load sounds from asset files when available, fall back to synthesized sounds

**File:** `frontend/src/app/home/services/sound.service.ts`

**Context:** Asset files exist in `frontend/src/assets/sounds/` for several actions. Currently, ALL sounds are synthesized via Web Audio API. The asset files should be used instead when available.

**Mapping of asset files to existing sound methods:**

  | Asset file              | Sound method to update    |
  |-------------------------|---------------------------|
  | `new_turn.ogg`          | `playNewTurn()`           |
  | `card.wav`              | `playCard()`              |
  | `capture.wav`           | `playCapture()`           |
  | `teleport.wav`          | `playSwap()`              |
  | `vitcory.wav`           | `playVictory()` (new)     |
  | `defeat.wav`            | `playDefeat()` (new)      |

**Implementation approach:**

  1. At service initialization (`constructor` or `ngOnInit`), attempt to `fetch()` each asset file.
  2. If the fetch succeeds, decode the audio data using `AudioContext.decodeAudioData()` and store the resulting `AudioBuffer` in a private map (keyed by sound name).
  3. In each `play*()` method, check if a pre-loaded `AudioBuffer` exists for that sound:
     - If yes: create a `BufferSource`, connect it to `ctx.destination`, and play it.
     - If no: fall back to the existing Web Audio API synthesis code.
  4. Handle potential fetch/decode errors gracefully (log a warning, fall back to synthesis).

**Note:** Asset loading uses `fetch()` with relative URLs like `'assets/sounds/new_turn.ogg'`. Ensure the Angular asset configuration in `angular.json` already includes the `sounds/` folder (it should by default under `assets/`).
