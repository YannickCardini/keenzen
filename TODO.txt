 5. Update player points in the database after each game outcome
--------------------------------------------------------------------------------

When a connected (non-guest) player's game ends, update their `points` field in
Cosmos DB. Apply the following point deltas:
  - Win:    +3
  - Loss:   -1
  - Abandon (disconnect while game is in progress): -2

Files:
  - backend/src/db.ts
  - backend/src/game/game.ts
  - backend/src/session/session-manager.ts
  - backend/src/auth/auth-router.ts  (if a dedicated endpoint is needed)

Steps:
  1. In db.ts, add a function:
       updateUserPoints(userId: string, delta: number): Promise<void>
     that increments the `points` field of the user document with the given
     userId (use Cosmos DB's patch or replace operation with $inc or equivalent).

  2. After updating points, call recomputeRankings() (see task 7) to keep the
     ranking field current.

  3. In game.ts, when the game ends (first player gets all marbles to arrival):
       - Identify which players are authenticated (have a Google userId, not a
         guest ID).
       - Call updateUserPoints(userId, +3) for the winner.
       - Call updateUserPoints(userId, -1) for each losing authenticated player.

  4. In session-manager.ts (or game.ts), detect when a WebSocket closes while
     the game is still in progress (player disconnects without game ending):
       - If the disconnected player is authenticated, call
         updateUserPoints(userId, -2).
       - Do not apply this penalty if the game had already ended naturally.

  5. Guest players (no Google account) are unaffected — do not attempt any DB
     update for them.

 7. Implement the ranking (leaderboard ladder) system
--------------------------------------------------------------------------------

A player's ranking is their integer position in the global leaderboard, ordered
by points descending. The player with the most points is ranked #1. Players with
equal points share the same rank (dense ranking).

The `ranking` field already exists on every user document in Cosmos DB.

Files:
  - backend/src/db.ts
  - backend/src/auth/auth-router.ts

Steps:
  1. In db.ts, implement:
       recomputeRankings(): Promise<void>
     - Fetch all users from the `users` collection, ordered by `points` DESC.
     - Assign integer ranks starting at 1 (use dense ranking: users with equal
       points get the same rank, the next rank is the next integer, not skipped).
     - Batch-update each user's `ranking` field in Cosmos DB.
     - This function must be called after every updateUserPoints() call (task 5).

  2. In auth-router.ts, add:
       GET /api/leaderboard
     Returns an array of users sorted by ranking ASC:
       [{ name, picture, points, ranking }, ...]
     Limit to the top N players (e.g., top 100) if the user base grows large.

  3. Ensure the user's current ranking is returned as part of the auth response
     (POST /api/auth/google response body) and/or via a GET /api/user/me
     endpoint so the frontend can display it in the profile panel (task 6).
