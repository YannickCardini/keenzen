Regrouper le bouton mute/unmute avec le bouton resign et game rules dans un sous-menu dans le table.component

**Contexte:**
- Le `SoundService` est dans `frontend/src/app/game/services/sound.service.ts`. Il n'a actuellement pas d'état mute — toutes les méthodes `play*()` synthétisent ou jouent l'audio sans condition.
- Le `table.component.html` possède déjà un bouton resign (`.resign-btn`, un bouton icône SVG) et un bouton game rules, définis dans `frontend/src/app/game/components/table/table.component.html`.
- Tous les sons du jeu passent par `SoundService`, injecté dans `board.component.ts` et `table.component.ts`.

**Tâches:**
1. Ajouter un signal `muted` (boolean, défaut `false`) dans `SoundService`. Au début de chaque méthode `play*()`, retourner immédiatement si `muted` vaut `true`.
2. Ajouter une méthode publique `toggleMute()` dans `SoundService`.
3. Créer un sous-menu dans `table.component.html` regroupant les trois actions :
   - Bouton mute/unmute (icône speaker-on / speaker-off selon l'état) — appelle `soundService.toggleMute()` au clic.
   - Bouton resign (déjà existant, à déplacer dans le sous-menu).
   - Bouton game rules (déjà existant, à déplacer dans le sous-menu).
4. Le sous-menu doit être cohérent visuellement (même style d'icônes, même taille et poids visuel que les boutons existants).
5. L'état mute n'a pas besoin de persister entre les rechargements de page (bonus : le stocker dans `localStorage`).
